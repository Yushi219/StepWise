你说得对，这版给 Claude Code 太容易被它理解成“每章同一个 pinned section + 换卡片”，结果做出来就像模板循环。我们要把它改成 **每章有完全不同的版式和动效机制**，同时保留最开始那版的清晰结构。

下面这版是给 Claude Code / Gemini 做 HTML 的 **导演式页面规格**：每章都写清楚布局、动效、素材、视觉目标，避免它复用同一种卡片模板。

---

# StepWise for Revit  
**Scrollytelling Case Study Layout Spec**

整体原则：  
每一章都使用不同的空间结构。不要重复“左文字右卡片”。页面应该像一条产品电影：有开场、证据墙、问题爆炸、系统重构、操作演示、收束。

---

## 00 · Opening Cinematic
**Purpose**  
建立主题：楼梯不是单段几何，而是一套跨楼层系统。

**Layout**  
Full-screen cinematic hero。  
文字极少，居中或左下角。

**Visual**  
全屏 3D stair model。  
从单个 stair run 近景拉远到 20 层 stair core。

**Animation**
- Scroll 0-30%：近景踏步，浅景深。
- Scroll 30-70%：相机后退，楼层逐渐出现。
- Scroll 70-100%：typical floors 蓝色、equipment floors 橙色亮起。

**Self-made assets**
- 单段楼梯近景
- 多层 stair core 模型 / AI 关键帧

**Foreground**
StepWise for Revit  
Edit globally. Vary locally. Preview impact.

## 01 · Existing Revit Context
**Purpose**  
说明 Revit 已有 creation 工具，StepWise 不是重做创建，而是解决创建后的编辑维护。

**Layout**  
Horizontal ribbon scan。  
顶部是一条超宽 Revit toolbar，页面像横向扫描 UI。

**Visual**  
你截图里的 Revit 2026 stair toolbar 横向铺开。  
Run、Landing、Support、Multistory Stairs 被 light sweep 高亮。

**Animation**
- 工具栏从右向左缓慢横移，像扫描产品界面。
- 高亮依次扫过 Run → Landing → Support → Multistory。
- 最后 toolbar 变半透明，后面浮现多层 stair model。

**Self-made assets**
- Revit 2026 toolbar 截图
- 模糊化/抽象化版本
- 多层楼梯模型

**Foreground**
Revit already supports stair creation.  
StepWise focuses on what happens after creation: change, exceptions, dependencies, and impact.

## 02 · Research Table / Evidence Desk
**Purpose**  
展示研究不是拍脑袋：case prompt、访谈、公开反馈、market scan 汇合。

**Layout**  
俯视研究桌面，不是卡片墙。  
像桌上散着打印纸、截图、便签、插件页面。

**Visual**
- PDF 片段打印纸
- 访谈便签
- 论坛截图纸片
- 插件 market scan 小票据
- 中央是一张多层楼梯草图

**Animation**
- 不同证据从桌面边缘滑入。
- 一支光线/细线把证据连接到中间 stair sketch。
- 最后桌面上只留下一个高价值场景图：repeated floors + exceptions + floor height change。

**Self-made assets**
- PDF 截图片段
- 模糊论坛卡片
- 插件扫描小卡片
- 中央楼梯手绘/模型图

**Foreground**
The pain is real, but contextual.  
It becomes severe when repeated floors, local exceptions, shared types, and project changes collide.

# 给 Claude Code 的关键限制

你可以直接把这段加到 prompt 里：

1. Do not use the same card layout for every section.  
2. Each section must have a distinct visual mechanism: cinematic hero, toolbar scan, evidence desk, timeline, exploded model, network cleanup, product UI reveal, vertical matrix, spotlight scope, branching diagram, ghost preview, side-by-side system comparison, interaction lab, test rig, final resolve.  
3. Background visuals should carry the story. Foreground text should not repeat the visual.  
4. Avoid generic SaaS cards. This is a BIM product case study, not a marketing landing page.  
5. Use Revit-like UI density and professional BIM visual language.  
6. No fake readable text in AI-generated background images; all real text must be HTML.  
7. Main UI mockups should look like tools a Revit user would trust: compact, structured, not playful.

这样它就不会再给你生成“每章卡片换一换”的东西了。
